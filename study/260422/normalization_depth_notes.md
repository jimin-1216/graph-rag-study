# 260422 Normalization Depth Notes

## 왜 이 문서를 남기나

파싱 과정에서 `조/항/호/목까지만 보면 된다`고 단정하기 어렵다는 점이 확인됐다.  
실제 문서군을 보면 규정, 지침, 검사매뉴얼, 별지서식, 부칙이 서로 다른 깊이와 표기 방식을 가진다.

따라서 `정규화 깊이 원칙`은 선언적으로 먼저 정하는 것이 아니라,

1. 문서군에서 구조가 어디까지 내려가는지 관찰하고
2. 그중 PoC에 필요한 깊이까지만 모델링하고
3. 그보다 깊은 구조는 원문 보존 + 후속 확장 대상으로 남기는 방식

으로 가져가는 것이 맞다.

## 오늘 확인한 구조 깊이

### 1. AML 내규(규정/지침)

핵심 내규 2종을 기준으로 현재 실제로 관찰된 구조는 다음과 같다.

- `편/장/절`
- `조`
- `항(①, ②, ③ ...)`
- `호(1., 2., 3. ...)`
- `목(가., 나., 다. ...)`
- `세부번호(1), 2), 3) ...)`
- `부칙`
- `별지서식`

즉, 문서에 따라 실제로는 `조 -> 항 -> 호 -> 목 -> 세부번호`까지 내려간다.

### 2. 검사매뉴얼

`(매뉴얼) 은행 검사매뉴얼('25.7.게시용).md`를 보면 일반 법령형 구조와 다르게 다음 패턴이 섞여 있다.

- `101`, `201`, `301` 같은 번호형 섹션
- `1.`, `2.`, `3.` 같은 절차/항목
- `가.`, `나.`, `다.` 같은 하위 설명
- 목차 페이지의 숫자열

즉, 검사매뉴얼은 법령형 조문 구조라기보다 `문단형/섹션형 운영 매뉴얼`에 가깝다.

### 3. 부칙/별지/표

실제 파싱 결과를 보면 다음이 일반 조문과 섞이면 품질이 급격히 떨어진다.

- `부칙`
- `별표`
- `별지서식`
- 이미지/표 기반 서식 본문

따라서 이들은 일반 `article` 계층과 분리하는 것이 맞다.

## 오늘 적용한 보정

핵심 내규 2종에 대해 다음 보정을 적용했다.

- `opendataloader` JSON 직해석 대신 Markdown(`.md`) 기준 룰 베이스 정규화로 전환
- `부칙`을 별도 `supplementary` 노드로 분리
- `별지서식`을 별도 `appendix` 노드로 분리
- `가/나/다`를 `subitem`으로 분리
- `1)/2)/3)`를 `detail_item`으로 분리

현재 내부 문서 정규화 결과는 다음과 같다.

- 문서: `2`
- 유닛: `654`
- 관계: `769`

현재 단위 타입:

- `heading`
- `article`
- `paragraph`
- `item`
- `subitem`
- `detail_item`
- `supplementary`
- `appendix`

## 관찰 결과

### 1. 모든 문서를 같은 계층으로 밀어넣으면 안 된다

법령/내규/검사매뉴얼은 문서 구조가 다르다.

- 법령/내규: 조문 구조 중심
- 검사매뉴얼: 운영 섹션/체크리스트 중심
- 별지/표: 서식/테이블 중심

즉, 하나의 보편 규칙으로 전부 정규화하려 하면 품질이 무너진다.

### 2. 구조를 끝까지 쪼개는 것은 비효율적이다

실제로는 `가)`, `나)`, 표 셀, 이미지 캡션, 부칙 내 세부 규정까지 계속 내려갈 수 있다.

하지만 현재 PoC 목표는 완전한 법제 DB가 아니라 `Graph RAG 입력 구조 안정화`다.  
따라서 깊이를 끝까지 따라가는 것은 비용 대비 효과가 낮다.

### 3. 그래도 "어디까지 내려가는지"는 파악해야 한다

모델링은 다 하지 않더라도, 문서군에 어떤 깊이가 존재하는지는 알아야 한다.

이유:

- 왜 파서가 깨지는지 설명할 수 있어야 한다
- 어디서 컷오프 할지 근거를 가져야 한다
- 특정 질의에서 더 깊은 구조가 필요한지 판단할 수 있어야 한다

## 현재 권장 컷오프

현 시점 PoC에서 우선 모델링할 깊이는 다음이 적절하다.

- `heading`
- `article`
- `paragraph`
- `item`
- `subitem`
- `detail_item`

별도 타입:

- `supplementary`
- `appendix`

후순위:

- `가)` 형태의 추가 하위 구조
- 표 내부 셀 단위 구조
- 이미지 기반 서식의 의미 구조
- 검사매뉴얼의 섹션형 번호 체계

즉 지금은 `목 + 1단계 세부번호`까지만 정규화하고, 그보다 깊은 구조는 원문 텍스트 보존으로 처리하는 것이 적절하다.

## 남은 과제

1. 전체 문서군 기준 구조 깊이 조사
- AML 내규 2종 외 나머지 AML 문서
- 검사매뉴얼
- 부칙/별지/표 구조

2. 문서 유형별 정규화 전략 분리
- 법령형 문서
- 내규형 문서
- 매뉴얼형 문서
- 서식형 문서

3. 컷오프 기준을 질의 시나리오와 연결
- 어떤 질문은 `article/item`까지만 있으면 충분한가
- 어떤 질문은 `subitem/detail_item`까지 필요로 하는가

## 결론

`어디까지 쪼개지는가`는 먼저 조사해야 한다.  
하지만 `쪼개질 수 있는 모든 깊이`를 전부 모델링할 필요는 없다.

이번 PoC에서는 문서군의 구조 깊이를 관찰한 뒤,

- 법령/내규는 `목 + 세부번호 1단계`까지
- 부칙/별지/표는 별도 타입
- 검사매뉴얼은 별도 정규화 전략

으로 가져가는 것이 현실적이다.
