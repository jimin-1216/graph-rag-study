# Graph RAG 파이프라인 정리

작성일: 2026-04-22

## 목적

기존 RAG에 그래프 탐색 레이어를 추가하여, 단순 문서 검색을 넘어 법령/내규 간 관계 추적이 가능한 구조를 정리한다.

이번 스터디에서는 특히 다음 질문을 풀 수 있는 구조를 목표로 한다.

- 특정 내부규정 조항의 상위법 근거는 무엇인가
- 어떤 의무가 누구에게 적용되는가
- 어떤 예외 또는 면제가 존재하는가
- 위반 시 어떤 제재가 연결되는가

## 핵심 아이디어

- 텍스트 리트리버는 원문과 chunk를 회수한다.
- 그래프 리트리버는 조문, 상위법, 예외, 적용대상, 제재 등의 관계를 추적한다.
- 두 결과를 통합해서 근거를 검증하고, 필요하면 한 번 더 탐색한다.
- LLM은 최종 정리와 답변 생성을 담당한다.

## 전체 구조

```text
[사용자 질문]
    |
    v
[질문 분석]
- 엔티티 추출
- 질문 유형 분류
- 필요한 탐색 깊이 판단
    |
    +---------------------------+
    |                           |
    v                           v
[텍스트 리트리버]            [그래프 리트리버]
- 벡터 검색                  - Neo4j 탐색
- 키워드/BM25               - 조/항/호/목 구조 추적
- 내부규정 chunk 회수         - 상위법/시행령/고시 연결
                             - 인용/참조/예외/대상/제재 탐색
    |                           |
    +-------------+-------------+
                  |
                  v
          [증거 통합 / 평가]
          - 출처 중복 제거
          - 법령/내규 연결 확인
          - 근거 부족 여부 판단
          - 충돌 여부 확인
                  |
          +-------+-------+
          |               |
        충분함          부족함
          |               |
          v               v
    [컨텍스트 조립]    [재탐색]
    - 핵심 조문          - 그래프 한 홉 더
    - 상위법 근거        - 다른 키워드로 재검색
    - 예외/대상/제재     - 관련 법령 추가 회수
    - provenance
          |
          v
        [LLM]
    - 근거 기반 답변 생성
    - 출처 포함
    - 불확실성 명시
          |
          v
       [최종 응답]
```

## AML / 법령 도메인에 맞춘 단순 버전

```text
질문
-> 엔티티 추출
-> 내부규정 검색
-> Neo4j에서 상위법/시행령/고시/인용조문 추적
-> 예외/적용대상/제재 연결
-> 근거 조문 + 원문 chunk 조립
-> LLM 답변
```

## 단계별 설명

### 1. 질문 분석

- 질문에서 법령명, 조문번호, 의무, 기관, 대상자 같은 핵심 엔티티를 뽑는다.
- 질문이 정의 확인형인지, 근거 추적형인지, 예외 확인형인지 구분한다.
- 몇 홉까지 그래프를 탐색할지 결정한다.

### 2. 텍스트 리트리버

- 내부규정 PDF에서 파싱된 chunk 또는 문단을 회수한다.
- 질문과 직접 관련된 문장과 조문 원문을 확보한다.
- LLM이 직접 읽을 텍스트 근거를 제공한다.

### 3. 그래프 리트리버

- Neo4j에서 관련 조문 노드를 출발점으로 탐색한다.
- `HAS_CHILD`를 통해 조/항/호/목 구조를 따라간다.
- `REFERS_TO`, `CITES`, `IMPLEMENTS` 계열 관계를 통해 상위법과 관련 문서를 찾는다.
- 예외, 적용대상, 제재와 연결된 의미 노드를 추적한다.

### 4. 증거 통합 / 평가

- 텍스트 검색과 그래프 탐색 결과를 합친다.
- 중복 근거를 정리하고, 출처가 부족한 결과를 걸러낸다.
- 질문에 답하기에 부족하면 한 번 더 탐색한다.

### 5. 컨텍스트 조립

- 핵심 조문 원문
- 상위법 근거
- 예외/적용대상/제재
- 관련 그래프 경로
- 출처 정보

위 정보를 답변용 컨텍스트로 정리한다.

### 6. LLM 답변 생성

- LLM은 회수된 근거를 요약하고 정리한다.
- 답변 시 근거 조문과 출처를 포함한다.
- 근거가 불충분한 경우 불확실성을 표시한다.

## 기존 RAG와의 차이

### 기존 RAG

- 질문과 유사한 문서를 찾는다.
- 관련 chunk를 모은다.
- LLM이 답을 합성한다.

### Graph RAG

- 질문에서 핵심 엔티티를 찾는다.
- 그래프에서 관계를 따라가며 관련 조문과 상위법을 찾는다.
- 텍스트와 그래프 결과를 함께 조립한다.
- 관계 기반 멀티홉 질문에 더 강하다.

## 이 구조를 쓰는 이유

- 법률/내규 도메인은 정의, 의무, 예외, 제재가 여러 조문에 흩어져 있다.
- 단순 유사도 검색만으로는 상위법 근거 추적이나 예외 연결이 약하다.
- Graph RAG는 문서 검색 결과를 구조적으로 보강해 준다.
- LLM이 추론을 담당하는 범위를 줄이고, 검색과 관계 추적을 앞단에서 처리할 수 있다.

## 이번 PoC 기준 최소 구현 범위

- 질문 분석: 엔티티 추출, 질문 유형 분류
- 텍스트 리트리버: 내부규정 chunk 회수
- 그래프 리트리버: 상위법, 조문 구조, 인용/참조 탐색
- 평가 단계: 근거 부족 여부 판단
- 컨텍스트 조립: 조문 원문 + 그래프 경로 + 출처
- LLM: 근거 기반 답변 생성

## 한 줄 정리

기존 RAG에 그래프 탐색기를 붙여, 원문은 텍스트 검색으로 찾고 관계는 Neo4j에서 추적하는 구조다.
