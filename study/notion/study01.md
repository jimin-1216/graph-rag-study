## 온톨로지, 지식그래프, 그래프 기본 개념

### 목표

- 온톨로지와 그래프를 “이론”이 아니라 “업무 개념 구조화 도구”로 이해

### 상세 아젠다

1. 온톨로지란 무엇인가
    - class / instance
    - property / relation
    - taxonomy
2. 지식그래프란 무엇인가
    - entity
    - relation
    - attribute
    - provenance
3. 온톨로지 vs 데이터 모델 vs taxonomy vs thesaurus
4. knowledge graph vs graph database
5. 법률·규제·정책 분야에서 왜 관계 모델이 중요한가
6. 실무 예시
    - 조문
    - 정의
    - 적용대상
    - 예외
    - 제재
    - 시행일

### 실습 과제

- 핵심 개념 사전 학습
- 샘플 규정 문서 1개를 보고 아래를 추출
    - 핵심 개체 10개
    - 핵심 관계 10개
- 화이트보드 또는 문서로 개념도 작성
- 관련 도구 조사 및 활용

### 산출물

- 도메인 개체 후보 리스트
- 관계 후보 리스트
- 초안 개념도

## 그래프 데이터 모델링과 질의 방식

### 목표

- 그래프를 어떻게 저장하고 질의하는지 이해
- 법률 문서 구조를 그래프로 바꿔보는 감 익히기

### 상세 아젠다

1. 그래프 모델 기초
    - node
    - edge
    - property
2. property graph 개념
3. RDF triple과 property graph의 차이
4. 그래프 질의 사고방식
    - 탐색
    - multi-hop
    - neighbor expansion
5. Cypher 개념
6. 법률 질의를 그래프 질의로 바꾸는 방법

### 실습 과제

- 아래 질문을 그래프 질의 관점으로 바꾸기
    - “이 조문이 참조하는 정의는 무엇인가?”
    - “이 의무의 적용 대상은 누구인가?”
    - “이 예외 규정과 연결되는 제재 조항은 무엇인가?”
- 간단한 샘플 그래프 스키마 작성
    - Provision
    - Definition
    - Obligation
    - Subject
    - Exception
    - Sanction

### 산출물

- 그래프 스키마 v0.1
- 예시 질의 목록
- 질의별 필요한 관계 매핑표
